Epicenter Education · Counsellor Flows

Full Counsellor Flow Storyboard — In PRD Order

Every use case from the PRD's Use Cases section (§4), in the order they're written, told as one continuous storyboard following the same small caseload of students. Static screens only — nothing here is clickable or functional. Counsellor view only, as scoped.

📌 Structure: one section per PRD use case, in the exact order listed in the PRD. Read each row left to right; the caption under a frame explains the action that produces the next frame.
🔄 v2 update: task and application statuses now follow a tick-then-confirm pattern; profiles gained Extracurriculars & Achievements and Test Scores & Academic History; evidence uploads show a real thumbnail; shortlist entries pair with a student priorities card (not an automated suggester); pop-ups now open centered; and a new Forms feature (UC10) was added using a Microsoft Forms embed.
✨ v3 update: AI features from `AI_Integrations_Spec_v1.md` and `AI_Integration_Flow_Plan_v1.md` are now woven directly into UC1 (Dashboard digest, note clean-up), UC2 (category-aware nudge, stalled-task alert), UC3 (risk flagging), UC5 (requirement checklist extraction), UC6 (reassignment handoff snapshot), UC7 (essay feedback first pass), and UC9 (meeting prep briefing) — look for the small AI badge.

Coverage check

All eight use cases from the PRD's Use Cases section (§4) are covered above, in the order they're written: seven as full screen-by-screen flows, and Pilot Rollout as the adoption note it actually is rather than a forced screen sequence. Use Case 9, Calendar & Google Sync, and Use Case 10, Creating a Form, were both added on request and are clearly marked as sitting outside the original PRD §4 list — the two points in this document where the interface goes further than what §8.11 Integrations currently commits to. Worth a short conversation on whether Google Calendar and Microsoft Forms become real V1 integrations alongside the existing Microsoft 365/OneDrive scope, or stay later additions, before either gets built.

Also updated in this v2 pass: every task and application requirement now follows a tick-then-confirm pattern (student marks it done, counsellor confirms before it counts as Complete); student profiles carry two new fields, Extracurriculars & Achievements and Test Scores & Academic History; evidence uploads show an actual thumbnail rather than a generic document placeholder; shortlist entries are paired with a student-set priorities card Rohan can reference (not an automated suggester); and every pop-up now opens centered on screen instead of pinned to the top right.
